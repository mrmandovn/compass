## Rules for filling this template

1. **No code** — no `code blocks`, no SQL, no API signatures, no class names. Plain language only.
2. **No TBD without owner** — if a section needs data you don't have, write "TBD — needs <stakeholder>" not just "TBD".
3. **Measurable goals** — every goal must have a way to verify it's achieved.
4. **Symmetric goals/non-goals** — don't put the same item in both lists.
5. **Concrete user flows** — happy path should be 5–8 specific steps, not generic "user uses feature".
